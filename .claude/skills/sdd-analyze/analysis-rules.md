# sdd-analyze 상세 분석 규칙

## 산출물 로드 규칙 (점진적 공개)

**spec.md에서:**
- 개요/컨텍스트
- 기능 요구사항
- 비기능 요구사항
- 사용자 스토리
- 엣지 케이스 (있는 경우)

**plan.md에서:**
- 아키텍처/기술 스택 선택
- 데이터 모델 참조
- 단계
- 기술적 제약사항

**tasks.md에서:**
- 태스크 ID
- 설명
- 단계 그룹화
- 병렬 마커 [P]
- 참조된 파일 경로

**constitution에서:**
- 원칙 검증을 위해 `.specify/memory/constitution.md` 로드

## 시맨틱 모델 구축

내부 표현을 생성합니다 (출력에 원본 산출물을 포함하지 마세요):

- **요구사항 인벤토리**: 각 기능 + 비기능 요구사항에 안정적인 키 부여 (명령형 구문에서 슬러그 도출; 예: "사용자가 파일을 업로드할 수 있다" → `user-can-upload-file`)
- **사용자 스토리/액션 인벤토리**: 수락 기준이 있는 개별 사용자 액션
- **태스크 커버리지 매핑**: 각 태스크를 하나 이상의 요구사항 또는 스토리에 매핑 (키워드/명시적 참조 패턴에 의한 추론)
- **constitution 규칙 세트**: 원칙 이름 및 MUST/SHOULD 규범적 명령문 추출

## 탐지 패스 상세

### A. 중복 탐지
- 유사 중복 요구사항 식별
- 품질이 낮은 표현을 통합 대상으로 표시

### B. 모호성 탐지
- 측정 가능한 기준이 없는 모호한 형용사 (빠른, 확장 가능한, 안전한, 직관적인, 견고한) 플래그
- 미해결 플레이스홀더 (TODO, TKTK, ???, `<placeholder>` 등) 플래그

### C. 미지정
- 동사는 있지만 목적어나 측정 가능한 결과가 없는 요구사항
- 수락 기준 정렬이 누락된 사용자 스토리
- spec/plan에 정의되지 않은 파일이나 컴포넌트를 참조하는 태스크

### D. constitution 정렬
- MUST 원칙과 충돌하는 요구사항 또는 계획 요소
- constitution에서 요구하는 필수 섹션 또는 품질 게이트 누락

### E. 커버리지 갭
- 연관된 태스크가 없는 요구사항
- 매핑된 요구사항/스토리가 없는 태스크
- 태스크에 반영되지 않은 비기능 요구사항 (예: 성능, 보안)

### F. 불일치
- 용어 표류 (파일 간 같은 개념이 다르게 명명됨)
- plan에서 참조되지만 spec에 없는 (또는 그 반대) 데이터 엔티티
- 태스크 순서 모순 (예: 종속성 주석 없이 기초 설정 태스크 전에 통합 태스크)
- 충돌하는 요구사항 (예: 하나는 Next.js를 요구하고 다른 하나는 Vue를 지정)

## 운영 원칙

### 컨텍스트 효율성
- **최소한의 고신호 토큰**: 실행 가능한 발견에 집중
- **점진적 공개**: 산출물을 점진적으로 로드
- **토큰 효율적 출력**: 발견 테이블을 50행으로 제한; 오버플로 요약
- **결정적 결과**: 변경 없이 재실행하면 일관된 ID와 수가 생성

### 분석 가이드라인
- **절대 파일을 수정하지 마세요** (읽기 전용 분석)
- **절대 누락된 섹션을 조작하지 마세요** (없으면 정확하게 보고)
- **constitution 위반을 우선시하세요** (항상 CRITICAL)
- **포괄적 규칙보다 예시를 사용하세요** (구체적 인스턴스 인용)
- **이슈가 없으면 우아하게 보고하세요** (커버리지 통계와 함께 성공 보고서)
