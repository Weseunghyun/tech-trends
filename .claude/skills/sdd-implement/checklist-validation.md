# 체크리스트 상태 확인

## FEATURE_DIR/checklists/가 존재하는 경우

checklists/ 디렉토리의 모든 체크리스트 파일을 스캔합니다.

### 각 체크리스트에 대해 계산

- 전체 항목: `- [ ]` 또는 `- [X]` 또는 `- [x]`와 일치하는 모든 라인
- 완료 항목: `- [X]` 또는 `- [x]`와 일치하는 라인
- 미완료 항목: `- [ ]`와 일치하는 라인

### 상태 테이블 생성

```text
| 체크리스트 | 전체 | 완료 | 미완료 | 상태 |
|-----------|------|------|--------|------|
| ux.md     | 12   | 12   | 0      | ✓ 통과 |
| test.md   | 8    | 5    | 3      | ✗ 실패 |
| security.md | 6  | 6    | 0      | ✓ 통과 |
```

### 전체 상태 결정

- **통과**: 모든 체크리스트에 미완료 항목이 0개
- **실패**: 하나 이상의 체크리스트에 미완료 항목이 있음

### 체크리스트가 미완료인 경우

1. 미완료 항목 수가 포함된 테이블 표시
2. **중단**하고 질문: "일부 체크리스트가 미완료입니다. 그래도 구현을 진행하시겠습니까? (예/아니오)"
3. 사용자 응답 대기
4. "아니오" → 실행 중단
5. "예" → 다음 단계로 진행

### 모든 체크리스트가 완료된 경우

- 통과 테이블 표시
- 자동으로 다음 단계로 진행
