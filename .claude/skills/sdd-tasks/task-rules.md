# 태스크 생성 규칙

**핵심**: 태스크는 독립적인 구현 및 테스트를 위해 사용자 스토리별로 조직되어야 합니다.

**테스트는 선택사항**: 기능 명세서에서 명시적으로 요청되거나 사용자가 TDD 접근법을 요청한 경우에만 테스트 태스크를 생성합니다.

## 체크리스트 형식 (필수)

모든 태스크는 반드시 다음 형식을 엄격히 따라야 합니다:

```text
- [ ] [TaskID] [P?] [Story?] 파일 경로가 포함된 설명
```

**형식 구성요소**:

1. **체크박스**: 항상 `- [ ]`로 시작 (마크다운 체크박스)
2. **태스크 ID**: 실행 순서대로 순차 번호 (T001, T002, T003...)
3. **[P] 마커**: 태스크가 병렬화 가능한 경우에만 포함 (다른 파일, 미완료 태스크에 종속성 없음)
4. **[Story] 라벨**: 사용자 스토리 단계 태스크에만 필수
   - 형식: [US1], [US2], [US3] 등 (spec.md의 사용자 스토리에 매핑)
   - 설정 단계: 스토리 라벨 없음
   - 기초 단계: 스토리 라벨 없음
   - 사용자 스토리 단계: 반드시 스토리 라벨 포함
   - 마무리 단계: 스토리 라벨 없음
5. **설명**: 정확한 파일 경로가 포함된 명확한 액션

**예시**:

- 올바름: `- [ ] T001 구현 계획에 따라 프로젝트 구조 생성`
- 올바름: `- [ ] T005 [P] src/middleware/auth.py에 인증 미들웨어 구현`
- 올바름: `- [ ] T012 [P] [US1] src/models/user.py에 User 모델 생성`
- 올바름: `- [ ] T014 [US1] src/services/user_service.py에 UserService 구현`
- 잘못됨: `- [ ] User 모델 생성` (ID와 Story 라벨 누락)
- 잘못됨: `T001 [US1] 모델 생성` (체크박스 누락)
- 잘못됨: `- [ ] [US1] User 모델 생성` (태스크 ID 누락)
- 잘못됨: `- [ ] T001 [US1] 모델 생성` (파일 경로 누락)

## 태스크 조직

1. **사용자 스토리에서 (spec.md)** - 기본 조직:
   - 각 사용자 스토리(P1, P2, P3...)가 자체 단계를 가짐
   - 관련 컴포넌트를 스토리에 매핑:
     - 해당 스토리에 필요한 모델
     - 해당 스토리에 필요한 서비스
     - 해당 스토리에 필요한 인터페이스/UI
     - 테스트 요청 시: 해당 스토리에 특화된 테스트
   - 스토리 종속성 표시 (대부분의 스토리는 독립적이어야 함)

2. **계약에서**:
   - 각 인터페이스 계약 → 서비스하는 사용자 스토리에 매핑
   - 테스트 요청 시: 각 인터페이스 계약 → 해당 스토리 단계에서 구현 전 계약 테스트 태스크 [P]

3. **데이터 모델에서**:
   - 각 엔티티를 필요한 사용자 스토리에 매핑
   - 엔티티가 여러 스토리에 걸치는 경우: 가장 이른 스토리 또는 설정 단계에 배치
   - 관계 → 적절한 스토리 단계의 서비스 레이어 태스크

4. **설정/인프라에서**:
   - 공유 인프라 → 설정 단계 (1단계)
   - 기초/차단 태스크 → 기초 단계 (2단계)
   - 스토리별 설정 → 해당 스토리 단계 내

## 단계 구조

- **1단계**: 설정 (프로젝트 초기화)
- **2단계**: 기초 (차단 전제조건 - 사용자 스토리 전에 반드시 완료)
- **3단계+**: 우선순위 순서의 사용자 스토리 (P1, P2, P3...)
  - 각 스토리 내: 테스트 (요청 시) → 모델 → 서비스 → 엔드포인트 → 통합
  - 각 단계는 완전하고 독립적으로 테스트 가능한 증분이어야 함
- **최종 단계**: 마무리 및 공통 관심사

## 보안 검증 태스크 자동 생성

`scripts/**`의 Python 코드를 생성/수정하는 기능이면, 마무리 단계에 보안/품질 검증 태스크를 포함한다:

- **검증 태스크**: `impl-python-validate` 스킬로 ruff(S-그룹)/bandit/pip-audit + 수동 보안 체크리스트(SEC-01~SEC-09)를 수행한다.
  - 예: `- [ ] T0NN impl-python-validate 실행하여 변경된 scripts 보안/품질 검증`

**주의**: 외부 입력(ticker 등)을 새로 다루거나 시크릿/외부 HTTP를 취급하는 태스크에는 `.claude/rules/python-security.md` 준수 확인을 완료 조건에 포함한다.
